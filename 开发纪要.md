# 单词好简答 · 开发纪要

## 一、项目概述

`单词好简答` 是一个面向小学生英语单词学习场景的微信小程序，目标是通过“背词 + 复习 + 游戏 + 成长反馈 + 自动听写”形成完整的学习闭环，降低家长陪学成本，提升孩子的主动学习动力。

本轮开发聚焦于 `V1.0 MVP` 的核心功能落地，并完成了从数据层、学习引擎、游戏化机制、听写模式，到本地多账号管理的整体打通。

---

## 二、开发目标

本阶段主要目标包括：

1. 建立原生微信小程序基础工程结构
2. 构建完整词库数据源与分级加载能力
3. 实现最小可用学习页与记忆曲线复习机制
4. 实现词库页、详情页与自动听写模式
5. 建立成长系统、金币系统、宠物互动系统
6. 完成拼字游戏闭环与无尽模式优化
7. 支持本地多账号管理与数据隔离
8. 完成 MVP 发布前的稳定性与体验打磨

### 6. 自动听写模式

已完成：
- 新建 `pages/dictation/index`
- 支持从词库页一键进入自动听写
- 沉浸式深色背景，弱化视觉干扰
- 不显示英文，仅显示中文释义
- 支持：
  - 开始 / 暂停
  - 上一个 / 下一个
  - 重复朗读当前单词
  - 语速 / 间隔设置

专业级优化：
- 听写定时逻辑改为：
  - 发音真正结束后
  - 再开始 N 秒倒计时
  - 倒计时结束自动切词
- 重复播放会重置倒计时
- 听写页支持 `wx.setKeepScreenOn`
- 全局配置了 `requiredBackgroundModes: ["audio"]`

该功能目标是：
- 彻底减轻家长人工报听写压力

关键体验优化：

## 四、关键技术与架构优化

### 1. 音频单例统一管理

已抽离 `utils/audio.js`，统一管理：
- `getSharedAudioContext()`
- `playWordAudio()`
- `stopWordAudio()`
- `setAudioEndedHandler()`
- `clearAudioEndedHandler()`

收益：
- 避免多页面重复创建音频实例
- 降低页面切换后音频失效风险
- 听写、学习、详情三类页面统一调用音频能力

### 2. 生命周期健壮性处理

已处理：
- 学习页音频 `onHide / onUnload` 安全停止
- 听写页后台保持与退出释放
- 小游戏自动切关定时器在 `onHide / onUnload` 中清理
- 多页面进入退出时状态重置更干净

### 3. 空状态与容错处理

已在以下页面增加空状态：
- 学习页
- 词库页
- 游戏页

避免：
- 词库为空时的白屏
- 无数据时的运行时异常
- 页面进入后没有内容但用户不知原因的问题

## 五、当前 V1.0 已形成的业务闭环

### 学习闭环
背词 → 认识 / 不认识 → 进入记忆曲线 → 复习 / 错题回流

### 成长闭环
背词 → 获得 EXP → 升级 → 获得成长反馈

### 金币闭环
完成学习 → 获得保底金币 → 小游戏连击赚金币 → 宠物消费金币

### 家长减负闭环
词库选择 → 一键进入自动听写 → 系统自动报词 → 孩子自主完成练习
